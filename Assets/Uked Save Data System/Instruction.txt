ğŸ“˜ AnhPV Save System - Documentation
A ScriptableObject-based save system using JSON serialization, optimized for Unity mobile development.

1. Core Components
SaveSystem.cs: Handles low-level I/O, JSON serialization, and optional GZip compression.
SaveDataSO<T>: Base class for data containers. Provides SaveData(), LoadData(), and DeleteData() methods.
BaseSaveDataEditor<T>: Custom Editor providing "Save", "Load", and "Delete" buttons directly in the Inspector.

2. Setup Workflow
Define Data Model: Create a plain C# class marked with [Serializable] (e.g., PlayerData).
Create ScriptableObject: Create a script inheriting from SaveDataSO<YourModel>.
Create Editor Script: Create a script inheriting from BaseSaveDataEditor<YourModel> to enable Inspector buttons.
Instantiate Asset: In Unity Editor, right-click -> Create > Game > Player Data (or your custom menu).

3. Usage in Code
Loading: Call _playerSO.LoadData() at game start (e.g., in DataManager.Awake).
Saving: Call _playerSO.SaveData() when data changes or during OnApplicationPause/OnApplicationQuit.
Accessing Data: Use DataManager.Instance.Player.coins to read/write values.

4. Advanced Configurations (SaveSystem.cs)
Compression: Set USE_COMPRESSION = true to enable GZip (smaller files, harder to manually edit).
Newtonsoft.Json:
Add HAS_NEWTONSOFT to Project Settings > Player > Scripting Define Symbols.
Set USE_NEWTONSOFT = true for faster and more robust serialization.

5. Important Notes
File Path: Data is stored at Application.persistentDataPath.
Serialization: Ensure all custom classes/structs inside your data model are marked [Serializable].
Mobile Stability: The DataManager automatically saves on OnApplicationPause, which is the most reliable way to prevent data loss on Android/iOS.
